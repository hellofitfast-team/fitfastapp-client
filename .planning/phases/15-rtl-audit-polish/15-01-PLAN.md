**Swipe direction (RTL-06):**
Find the check-in wizard component that uses react-swipeable. In RTL mode (Arabic locale), swipe directions must be inverted:
- In LTR: swipe left = next step, swipe right = previous step
- In RTL: swipe right = next step, swipe left = previous step

Implementation approach:
1. Use `useLocale()` from next-intl to detect current locale
2. Derive `isRTL = locale === 'ar'`
3. In the swipe handler, swap the `onSwipedLeft` and `onSwipedRight` callbacks when `isRTL` is true
4. OR invert the direction check: `const nextDirection = isRTL ? 'Right' : 'Left'`

**Day selector scroll (RTL-07):**
Find the horizontal day selector component used in meal plan and workout plan pages. When in RTL mode:
- Day 1 should appear on the RIGHT edge (not left)
- The active/current day should scroll into view from the right side
- Initial scroll position should be at the end (rightmost) in RTL

Implementation:
1. After mount (useEffect), check if RTL
2. If RTL, set `scrollLeft` to `scrollWidth - clientWidth` (scroll to end) or use `scrollTo({ left: scrollWidth })` — the browser handles RTL scrollLeft direction
3. When scrolling to active day, use `element.scrollIntoView({ inline: 'nearest', block: 'nearest' })` which respects RTL automatically
4. Test that day numbers still read 1, 2, 3... from right-to-left in Arabic mode

Ensure both components use the locale from next-intl (not hardcoded direction).

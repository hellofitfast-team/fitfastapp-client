  <action>
**PWA Manifest verification:**
1. Read `apps/client/public/manifest.json` — confirm `"display": "standalone"` is set
2. Confirm `theme_color` and `background_color` match the app's design tokens
3. Confirm icons are referenced and exist

**Safe area CSS setup:**
1. In the root layout or globals.css, ensure the viewport meta tag includes `viewport-fit=cover`:
   ```html
   <meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover" />
   ```
   This enables safe area inset env variables on iOS.

2. In `globals.css`, add safe area CSS custom properties if not present:
   ```css
   :root {
     --safe-area-top: env(safe-area-inset-top, 0px);
     --safe-area-bottom: env(safe-area-inset-bottom, 0px);
     --safe-area-left: env(safe-area-inset-left, 0px);
     --safe-area-right: env(safe-area-inset-right, 0px);
   }
   ```

**Component safe area handling:**
3. **Bottom navigation**: Must have bottom padding that accounts for the home indicator on iPhone:
   - Add `pb-[env(safe-area-inset-bottom)]` or use the CSS variable `pb-[var(--safe-area-bottom)]`
   - The bottom nav should NOT be clipped by the home indicator bar

4. **Mobile header**: Must account for the status bar / notch area:
   - Add `pt-[env(safe-area-inset-top)]` or use the CSS variable
   - Content should not sit behind the status bar in standalone mode

5. **Dashboard layout**: The main content area should have appropriate padding to avoid bottom nav overlap:
   - Confirm `pb-` accounts for both the bottom nav height AND the safe area

6. **Body/html**: Add `min-h-[100dvh]` (dynamic viewport height) to handle mobile browser chrome correctly. Use `dvh` not `vh` to prevent the 100vh bug on mobile Safari.

Do NOT add safe area handling to:
- Desktop-only components (these don't have notches/home indicators)
- Modal/drawer overlays (these are already full-screen with their own handling from Vaul)
  </action>

Verify safe area usage: `grep -r "safe-area-inset" apps/client/src/` — should find results in bottom nav and header.
  </verify>

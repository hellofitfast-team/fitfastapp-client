  <action>
Audit ALL interactive elements in renovated components for 44px minimum touch targets (VERIFY-02). The existing v1.0 audit (Phase 06) established 48px targets, but new components from Phases 11-14 may not follow this.

**Elements to check:**
- Buttons (including icon buttons, close buttons, toggle buttons)
- Links that look like buttons or nav items
- Bottom navigation items
- Tab selectors and day selector items
- Form inputs, checkboxes, toggles
- Expandable/collapsible card triggers
- Swipe step indicators
- Bottom sheet menu items

**Minimum sizing rules:**
- Standalone buttons: `min-h-11` (44px) or larger (`h-12` = 48px preferred per v1.0)
- Icon-only buttons: `min-h-11 min-w-11` (44px x 44px) minimum
- Nav items: `min-h-11` with adequate padding
- List items that are tappable: `min-h-11` or `py-3` minimum
- Inline links: acceptable at text size if surrounded by adequate whitespace

**For each undersized element:**
1. Add `min-h-11` (for height) or `min-w-11` (for width) or increase padding
2. Prefer `h-12 w-12` for icon buttons (per v1.0 convention)
3. Ensure the visual design still looks balanced after size increase

Do NOT increase sizes for:
- Non-interactive decorative elements (badges, status indicators, icons)
- Elements already meeting 44px+ (check computed size from classes)
  </action>
  <verify>
Search for small interactive elements: `grep -rn "h-8\\|h-9\\|h-10\\|w-8\\|w-9\\|w-10" apps/client/src/ apps/admin/src/ --include="*.tsx"` — review any remaining small sizes to confirm they are non-interactive.

Run: `pnpm type-check` — no TypeScript errors.
  </verify>
  <done>All interactive elements in renovated components meet 44px minimum touch target. Icon buttons use h-12 w-12 per convention.</done>

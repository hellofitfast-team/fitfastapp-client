Convert the onboarding initial assessment from a single long scrollable form into a guided step-by-step wizard with one section per screen, swipe navigation, and a segmented progress bar.

Purpose: New users completing their first assessment should experience a focused, guided flow rather than an overwhelming wall of fields. One-question-per-screen reduces cognitive load and matches the check-in wizard UX.
Output: A 5-step assessment wizard (Goals, Body Info, Schedule, Diet, Medical) with step navigation, swipe support, progress indicator, and per-step validation.
</objective>

  <action>
    Refactor the `InitialAssessmentPage` to show one section per screen with step navigation and swipe support. Keep ALL existing useState hooks and section components -- only change the rendering to be step-based.

    **Step structure (5 steps):**
    1. Goals (GoalsSection)
    2. Body Info (BasicInfoSection -- weight, height, experience, equipment)
    3. Schedule (ScheduleSection)
    4. Diet (DietarySection -- food prefs, allergies, restrictions)
    5. Medical (MedicalSection) + Submit button

    **Changes to page.tsx:**

    1. Add imports:
       ```typescript
       import { useSwipeable } from "react-swipeable";
       import { AssessmentProgress } from "./_components/assessment-progress";
       import { ArrowLeft } from "lucide-react"; // Add to existing lucide import
       ```

    2. Add step state and step config:
       ```typescript
       const [currentStep, setCurrentStep] = useState(1);
       const TOTAL_STEPS = 5;
       const STEP_LABELS = [
         t("steps.goals"),
         t("steps.bodyInfo"),
         t("steps.schedule"),
         t("steps.diet"),
         t("steps.medical"),
       ];
       ```
       Note: These translation keys need to exist. Add fallback logic: if the translation key doesn't exist, use hardcoded English labels as the default. The executor should add the translation keys to the client EN and AR message files under `onboarding.assessment.steps.*`.

    3. Add per-step validation function:
       ```typescript
       const validateStep = (step: number): string | null => {
         switch (step) {
           case 1: {
             const finalGoals = getFinalValues(selectedGoals, goalsOther);
             if (finalGoals.length === 0) return tErrors("goalRequired");
             return null;
           }
           case 2: {
             if (!currentWeight || !height) return tErrors("weightHeightRequired");
             if (!experienceLevel) return tErrors("experienceLevelRequired");
             if (!equipment) return tErrors("equipmentRequired");
             if (equipment === "other" && !equipmentOther.trim()) return tErrors("equipmentSpecify");
             return null;
           }
           case 3: {
             if (selectedDays.length === 0) return tErrors("workoutDaysRequired");
             return null;
           }
           case 4: return null; // Dietary is optional
           case 5: return null; // Medical is optional
           default: return null;
         }
       };
       ```

    4. Add step navigation handlers:
       ```typescript
       const handleNext = () => {
         const validationError = validateStep(currentStep);
         if (validationError) {
           setError(validationError);
           return;
         }
         setError(null);
         if (currentStep < TOTAL_STEPS) {
           setCurrentStep(currentStep + 1);
         }
       };

       const handleBack = () => {
         setError(null);
         if (currentStep > 1) {
           setCurrentStep(currentStep - 1);
         }
       };
       ```

    5. Add swipe handlers (same pattern as check-in from plan 14-01):
       ```typescript
       const swipeHandlers = useSwipeable({
         onSwipedLeft: () => {
           const isRtl = document.dir === 'rtl';
           if (isRtl) handleBack();
           else handleNext();
         },
         onSwipedRight: () => {
           const isRtl = document.dir === 'rtl';
           if (isRtl) handleNext();
           else handleBack();
         },
         delta: 50,
         preventScrollOnSwipe: false,
         trackTouch: true,
         trackMouse: false,
       });
       ```

    6. Rewrite the form render to show one section at a time:
       ```tsx
       <div className="max-w-3xl mx-auto space-y-6 pb-12">
         {/* Header */}
         <div className="text-center">
           <h1 className="text-3xl font-bold tracking-tight">{t("title")}</h1>
           <p className="mt-2 text-sm text-muted-foreground">{t("subtitle")}</p>
         </div>

         {/* Progress Bar */}
         <AssessmentProgress
           currentStep={currentStep}
           totalSteps={TOTAL_STEPS}
           stepLabels={STEP_LABELS}
         />

         <form onSubmit={onSubmit} className="space-y-6">
           {error && (
             <div className="rounded-lg border border-error-500/30 bg-error-500/10 p-4">
               <p className="text-sm font-medium text-error-500">{error}</p>
             </div>
           )}

           {/* Step content with swipe */}
           <div {...swipeHandlers} className="touch-pan-y">
             {currentStep === 1 && (
               <GoalsSection ... />
             )}
             {currentStep === 2 && (
               <BasicInfoSection ... />
             )}
             {currentStep === 3 && (
               <ScheduleSection ... />
             )}
             {currentStep === 4 && (
               <DietarySection ... />
             )}
             {currentStep === 5 && (
               <MedicalSection ... />
             )}
           </div>

           {/* Navigation */}
           <div className="flex items-center justify-between gap-3">
             <Button
               type="button"
               variant="outline"
               onClick={handleBack}
               disabled={currentStep === 1 || isLoading}
             >
               <ArrowLeft className="h-4 w-4 rtl:rotate-180" />
               {tCommon("back")}
             </Button>

             {currentStep < TOTAL_STEPS ? (
               <Button type="button" variant="gradient" onClick={handleNext} disabled={isLoading}>
                 {tCommon("next")}
                 <ArrowRight className="h-5 w-5 rtl:rotate-180" />
               </Button>
             ) : (
               <Button type="submit" variant="gradient" size="lg" disabled={isLoading} loading={isLoading}>
                 {isLoading ? t("submitting") : (
                   <>
                     {t("completeAssessment")}
                     <ArrowRight className="h-5 w-5 rtl:rotate-180" />
                   </>
                 )}
               </Button>
             )}
           </div>
         </form>
       </div>
       ```

    7. Remove the full-form validation from `onSubmit` since per-step validation now handles it. The `onSubmit` only fires from the last step. Still keep the submission logic (submitAssessment, updateProfile, generateMealPlan, generateWorkoutPlan) exactly as-is.

    8. Add transition class to step content: wrap each section render in a div with `className="animate-fade-in"` for entrance animation. If the animate-fade-in utility from Phase 12 is not available, add a simple inline style fallback: `style={{ animation: 'fadeIn 0.2s ease-out' }}`.

    9. Add translation keys to EN and AR message files for the step labels:
       - EN: `apps/client/messages/en.json` under `onboarding.assessment.steps`: `{ "goals": "Goals", "bodyInfo": "Body Info", "schedule": "Schedule", "diet": "Diet", "medical": "Medical" }`
       - AR: `apps/client/messages/ar.json` under `onboarding.assessment.steps`: `{ "goals": "الأهداف", "bodyInfo": "معلومات الجسم", "schedule": "الجدول", "diet": "التغذية", "medical": "طبي" }`
       - Also add `onboarding.assessment.completeAssessment` and `onboarding.assessment.submitting` if they don't exist (check first).

    **IMPORTANT:** Do NOT change any of the existing section components (GoalsSection, BasicInfoSection, etc.). They receive the same props as before. Only the page.tsx rendering changes.

    **IMPORTANT:** Keep all existing useState hooks in page.tsx. State lives in the parent and persists across step changes because React preserves parent state even when children unmount.

**1. Max-width constraint (DESK-02):**
Every page's root wrapper should have `max-w-3xl mx-auto` (for content-heavy pages like tickets, FAQ, settings) or `max-w-5xl mx-auto` (for grid-heavy pages like home, tracking, progress). Apply the appropriate constraint:

| Page | Max Width | Reasoning |
|------|-----------|-----------|
| Home (page.tsx) | max-w-5xl | Grid layout with multiple cards |
| Meal plan | max-w-3xl | Single-column content with day selector |
| Workout plan | max-w-3xl | Single-column content with day selector |
| Tracking | max-w-5xl | Multi-section with sub-components |
| Progress | max-w-5xl | Stats grid + charts |
| Tickets list | max-w-3xl | Single-column list |
| Ticket detail | max-w-3xl | Chat conversation column |
| Settings | max-w-3xl | Form content |
| FAQ | max-w-3xl | Accordion list |

For each page, ensure the root wrapper div has: `className="px-4 lg:px-6 py-6 max-w-{size} mx-auto"` (or equivalent).

**2. Responsive card grids (DESK-03):**

**Home page:**
- Desktop grid already handled by Plan 01 (`hidden lg:grid lg:grid-cols-2 gap-4`)
- Verify the quick stats section uses `grid grid-cols-2 lg:grid-cols-4` on desktop

**Tracking page:**
- Sub-component cards should flow in a 2-column grid on desktop: `lg:grid lg:grid-cols-2 gap-4` for the meal-tracking and workout-tracking cards side by side

**Progress page:**
- Stats overview: `grid grid-cols-2 lg:grid-cols-4 gap-3`
- Chart section: full width on all viewports (charts need horizontal space)

**3. Horizontal padding consistency:**
- Mobile: `px-4` (16px each side)
- Desktop: `lg:px-6` (24px each side) — adds a bit more breathing room on wide screens
- Check that no page uses `px-0` or inconsistent padding

**4. Day selector on desktop:**
- Meal plan and workout plan day selectors should not stretch full-width on desktop
- Update the DaySelector component (`day-selector.tsx`): add `lg:mx-0 lg:px-0` to its wrapper to prevent the `-mx-4 px-4` negative margin bleed on desktop viewports

**5. Ticket conversation on desktop:**
- Chat bubbles should maintain `max-w-[75%]` constraint even on desktop (already set — verify)
- The reply area should respect the max-w-3xl container

**6. Settings on desktop:**
- Section cards should stack vertically (single column is appropriate for forms)
- No grid needed — just verify max-width constraint

**7. FAQ on desktop:**
- Single column is appropriate — verify max-width constraint
- Search input should not stretch too wide: `max-w-md` on the search input

For each file: read current state, identify what's missing, and add only the necessary responsive classes. Do NOT restructure any page — only add/fix responsive utility classes.

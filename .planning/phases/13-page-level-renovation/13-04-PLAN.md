Rewrite the conversation view while preserving all existing data hooks, reply form logic, and back navigation.

**1. Message grouping by date:**
Create a helper function `groupMessagesByDate(messages)` that:
- Groups messages by calendar day (using `toISOString().split('T')[0]`)
- Returns `Array<{ dateKey: string; label: string; messages: Message[] }>`
- Label logic: if same day as today → `t("tickets.chat.today")`, if yesterday → `t("tickets.chat.yesterday")`, else → `new Date(dateKey).toLocaleDateString(locale, { month: 'short', day: 'numeric' })`
- Use `useLocale()` for locale-aware date formatting

**2. Date separator component:**
Between each date group, render:
```tsx
<div className="flex items-center gap-3 my-4">
  <div className="flex-1 h-px bg-border" />
  <span className="text-xs text-muted-foreground font-medium px-2 py-1 bg-neutral-100 rounded-full">
    {label}
  </span>
  <div className="flex-1 h-px bg-border" />
</div>
```

**3. Chat bubble rendering:**
For each message, determine if it's from coach (check `message.senderRole === "coach"` or similar field — inspect existing data structure).

**Coach message (left/gray):**
```
<div className="flex items-end gap-2 justify-start">
  <div className="flex h-7 w-7 shrink-0 items-center justify-center rounded-full bg-neutral-200 mb-1">
    <Shield className="h-3.5 w-3.5 text-muted-foreground" />
  </div>
  <div className="max-w-[75%] rounded-2xl rounded-bl-sm bg-neutral-100 text-foreground px-4 py-2.5 shadow-sm">
    <p className="text-sm leading-relaxed whitespace-pre-wrap">{message.message}</p>
    <p className="text-[10px] text-muted-foreground mt-1">{formatTime(message.createdAt)}</p>
  </div>
</div>
```

**Client message (right/blue):**
```
<div className="flex items-end gap-2 justify-end">
  <div className="max-w-[75%] rounded-2xl rounded-br-sm bg-primary text-primary-foreground px-4 py-2.5 shadow-sm">
    <p className="text-sm leading-relaxed whitespace-pre-wrap">{message.message}</p>
    <p className="text-[10px] text-primary-foreground/70 mt-1">{formatTime(message.createdAt)}</p>
  </div>
</div>
```

**RTL note:** The `justify-start` / `justify-end` flex direction is automatically RTL-safe. However, the bubble tail corners (`rounded-bl-sm` for coach, `rounded-br-sm` for client) are NOT RTL-safe. Use conditional classes:
- Coach: `isRtl ? "rounded-br-sm" : "rounded-bl-sm"`
- Client: `isRtl ? "rounded-bl-sm" : "rounded-br-sm"`
Where `isRtl = locale === "ar"` via `useLocale()`.

Send button: `h-10 w-10 rounded-full bg-primary text-primary-foreground flex items-center justify-center active:scale-[0.97]`.

**6. Scroll behavior:**

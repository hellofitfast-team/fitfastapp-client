    Read each file and replace hardcoded hex values with semantic Tailwind classes. Apply these mapping rules consistently:

    **Brand/Primary accent (orange -> Royal Blue):**
    - `bg-[#FF3B00]` -> `bg-primary` (primary brand backgrounds: section headers, CTA buttons, icon backgrounds)
    - `text-[#FF3B00]` -> `text-primary` (primary brand text: highlighted names, accents)
    - `border-[#FF3B00]` -> `border-primary` (primary brand borders)
    - `hover:bg-[#FF3B00]` -> `hover:bg-primary` (hover states on buttons/links)
    - `hover:text-[#FF3B00]` -> not expected in these files
    - `hover:border-[#FF3B00]` -> `hover:border-primary`
    - `bg-[#FF3B00]/10` -> `bg-primary/10` (light primary backgrounds)
    - `ring-[#FF3B00]` -> `ring-primary`

    **Error-specific uses of orange (keep as error):**
    - Error messages: `text-[#FF3B00]` when inside error display context (e.g., next to error icon, validation messages) -> `text-error-500`
    - Error borders: `border-[#FF3B00]` when wrapping error content -> `border-error-500`
    - Error backgrounds: `bg-[#FF3B00]/10` when showing error state -> `bg-error-500/10`
    - Password strength "weak" indicator: `bg-[#FF3B00]` -> `bg-error-500`

    **How to distinguish brand vs error for #FF3B00:**
    - If the element is a CTA button, section header, hover state, or brand accent -> `primary`
    - If the element displays error text, validation message, error border, or destructive state -> `error-500`
    - In set-password/page.tsx: the error display divs use `border-[#FF3B00] bg-[#FF3B00]/10` with error text -> `border-error-500 bg-error-500/10`
    - In set-password/page.tsx: password strength level 1 bar `bg-[#FF3B00]` -> `bg-error-500` (weak), `text-[#FF3B00]` for weak text -> `text-error-500`
    - In set-password/page.tsx: the success state `bg-[#00FF94]` header -> `bg-success-500`

    **Brand/Accent green -> primary or success:**
    - `bg-[#00FF94]` for icon containers, brand accents, active states -> `bg-primary` (these become Royal Blue as the new accent)
    - `text-[#00FF94]` for "FF" logo text, tagline text, decorative labels -> `text-primary` (brand accent text)
    - `hover:bg-[#00FF94]` -> `hover:bg-primary`
    - `hover:text-[#00FF94]` -> `hover:text-primary`
    - `bg-[#00FF94]/10` -> `bg-primary/10`

    **Success-specific uses of green (keep as success):**
    - Success confirmation screens (set-password success, magic-link sent): `bg-[#00FF94]` section header -> `bg-success-500`
    - Check icons in success states: `text-[#00FF94]` -> `text-success-500`
    - Password strength "strong" bar: `bg-[#00FF94]` -> `bg-success-500`

    **How to distinguish brand vs success for #00FF94:**
    - If the element is a success confirmation, completion indicator, or positive feedback -> `success-500`
    - If the element is a brand accent, logo element, tagline, icon background, or interactive state -> `primary`

    **Cream and black (mostly keep):**
    - `bg-[#FFFEF5]` -> `bg-cream`
    - `text-[#FFFEF5]` -> `text-cream`
    - `hover:text-[#FFFEF5]` -> `hover:text-cream`
    - `#000000` in inline Tailwind classes is fine as `black` (Tailwind default)
